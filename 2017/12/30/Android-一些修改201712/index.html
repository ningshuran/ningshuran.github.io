[QUESTION]

[耳机]视频播放界面长按耳机控制键不会进入语音搜索界面

[ANSWER]

对比Youtebu、GMS包的Photos耳机长按都可以进入语音搜索界面，而此视频播放界面是骁龙图库的中的，搜索源码并没有对耳机长按的操作。

adb shell

getevent -l 查看耳机按下的键为

getevent 可以查看对应键值

经过十六进制转换成十进制，得出键值为226
在项目中搜索226，查看对应的 .kl可知道此键为

\frameworks\base\data\keyboards\Generic.kl

key 226 HEADSETHOOK

再查看

\frameworks\base\core\java\android\view\KeyEvent.java
搜索“HEADSETHOOK”
可得

public static final int KEYCODE_HEADSETHOOK = 79;

所以可在视频播放应用中添加onKeyLongPress()处理长按，但如何弹出语音搜索界面是个问题，

故干脆在
\frameworks\base\services\core\java\com\android\server\policyPhoneWindowManager.java 直接添加算了

[音乐]添加音乐小部件后，点击小部件播放音乐后暂停，等待3分钟后不显示歌曲名

[ANSWER]

日志中可见打印了Action: android.media.action.CLOSE_AUDIO_EFFECT_CONTROL_SESSION的广播

在\packages\apps\SnapdragonMusic\src\com\android\music\MediaPlaybackService.java

中走onDestroy()时 会发送

Intent i = new Intent(AudioEffect.ACTION_CLOSE_AUDIO_EFFECT_CONTROL_SESSION);

此时会释放所有MediaPlayer资源，包括本地播放器和唤醒锁

mPlayer.release(); 
mPlayer = null;

之后还会发送更新小部件的广播

Intent intent = new Intent(UPDATE_WIDGET_ACTION);
sendBroadcast(intent);

针对这个广播的处理在

packages\apps\SnapdragonMusic\src\com\android\music\MediaAppWidgetProvider.java

@Override
public void onReceive(Context context, Intent intent) {
    String action = intent.getAction();
    if (action.equals(UPDATE_WIDGET_ACTION) && hasInstances(context)) {
        //receive from MediaPlaybackService onDestroy
        final RemoteViews views = new RemoteViews(context.getPackageName(), mWidgetLayoutId);
        AppWidgetManager appWidgetManager = AppWidgetManager.getInstance(context);
        int[] appWidgetIds = appWidgetManager.getAppWidgetIds(new ComponentName(context, this.getClass()));
        views.setViewVisibility(R.id.trackname, View.GONE);
        //此处默认的显示文本是“触摸可选择音乐。”
        views.setTextViewText(R.id.artist, context.getText(R.string.widget_initial_text));
        views.setViewVisibility(R.id.album, View.GONE);
        views.setImageViewResource(R.id.icon, R.drawable.album_cover_background);
        views.setImageViewResource(R.id.pause, R.drawable.notification_play);
        /* everytime pushUpdate(updateAppWidget) should do linkButtons, otherwise the buttons will not work */
        linkButtons(context, views, false /* not playing */);
        pushUpdate(context, appWidgetIds, views);
        //this time the service is died,when service reactivate no need call performUpdate
        //otherwise title will be falsh
        mNoneedperformUpdate = true;
    }
    super.onReceive(context, intent);
}

而“”为默认显示字段，故当资源释放后会不显示歌曲名，只显示默认字段。此现象的发生是正常，此为正常的系统资源回收的结果，如果音乐驻留后台是不会出现此现象。

[SOLUTION]

可以不做修改，也可在onDestroy()时删除发送更新小部件的广播

public void onDestroy() {
    if (!mIsReadGranted)
        return;
    // Check that we're not being destroyed while something is still playing.
    if (isPlaying()) {
        Log.e(LOGTAG, "Service being destroyed while still playing.");
    }
    // release all MediaPlayer resources, including the native player and wakelocks

    Intent i = new Intent(AudioEffect.ACTION_CLOSE_AUDIO_EFFECT_CONTROL_SESSION);
    i.putExtra(AudioEffect.EXTRA_AUDIO_SESSION, getAudioSessionId());
    i.putExtra(AudioEffect.EXTRA_PACKAGE_NAME, getPackageName());
    sendBroadcast(i);
    //释放所有MediaPlayer资源，包括本地播放器和唤醒锁

    mPlayer = null;

    mAudioManager.unregisterRemoteControlClient(mRemoteControlClient);
