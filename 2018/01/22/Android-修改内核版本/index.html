Android-修改内核版本

[QUESTION]
Kernel version displayed “编译用户@编译主机” name inside it

[ANSWER]

在第三方APP（CPU X）中显示的内核版本信息为

Linux version 3.10.49-g4f81c1e (编译用户@编译主机) (gcc version 4.8 (GCC) ) #2 SMP PREEMPT Mon Jan 22 01:51:29 EST 2018

而设置->关于 中显示的内核版本与之不一样

设置中显示是在

\frameworks\base\packages\SettingsLib\src\com\android\settingslib\DeviceInfoUtils.java

public static String formatKernelVersion(String rawKernelVersion) {
    // Example (see tests for more):
    // Linux version 3.0.31-g6fb96c9 (android-build@xxx.xxx.xxx.xxx.com) \
    //     (gcc version 4.6.x-xxx 20120106 (prerelease) (GCC) ) #1 SMP PREEMPT \
    //     Thu Jun 28 11:02:39 PDT 2012

    final String PROC_VERSION_REGEX =
            "Linux version (\\S+) " + /* group 1: "3.0.31-g6fb96c9" */
            "\\((\\S+?)\\) " +        /* group 2: "x@y.com" (kernel builder) */
            "(?:\\(gcc.+? \\)) " +    /* ignore: GCC version information */
            "(#\\d+) " +              /* group 3: "#1" */
            "(?:.*?)?" +              /* ignore: optional SMP, PREEMPT, and any CONFIG_FLAGS */
            "((Sun|Mon|Tue|Wed|Thu|Fri|Sat).+)"; /* group 4: "Thu Jun 28 11:02:39 PDT 2012" */

    Matcher m = Pattern.compile(PROC_VERSION_REGEX).matcher(rawKernelVersion);
    if (!m.matches()) {
        Log.e(TAG, "Regex did not match on /proc/version: " + rawKernelVersion);
        return "Unavailable";
    } else if (m.groupCount() < 4) {
        Log.e(TAG, "Regex match on /proc/version only returned " + m.groupCount()
                + " groups");
        return "Unavailable";
    }
    //此处读取的是配置文件中的ro.product.brand
    String brandName = SystemProperties.get("ro.product.brand", "Panasonic");
    //返回了一串字符用于显示内核版本
    return m.group(1) + "\n" +                 // 3.0.31-g6fb96c9
            brandName + " " + m.group(3) + "\n" + // x@y.com #1 
            m.group(4);                            // Thu Jun 28 11:02:39 PDT 2012
}

SystemProperties.get(“ro.product.brand”, “Panasonic”)拿到的是build.prop

所以第三方APP肯定不是这样获取的，而第三方更为可靠的获取方式是可以读取手机中proc/version，version文件完整显示了内核版本信息

Linux version 3.10.49-g4f81c1e (编译用户@编译主机) (gcc version 4.8 (GCC) ) #2 SMP PREEMPT Mon Jan 22 01:51:29 EST 2018

[SOLUTION]

要修改version中内核版本信息
可以搜索到

尝试修改其中LINUX_COMPILE_BY可以达到效果，但已有注释，最好不要动这里，因为这样有可能导致内内核版本显示不可用

所以最终修改
\kernel\scripts\mkcompile.h

if [ -z "$KBUILD_BUILD_VERSION" ]; then
    if [ -r .version ]; then
        VERSION=`cat .version`
    else
        VERSION=0
        echo 0 > .version
    fi
else
    VERSION=$KBUILD_BUILD_VERSION
fi

if [ -z "$KBUILD_BUILD_TIMESTAMP" ]; then
    TIMESTAMP=`date`
else
    TIMESTAMP=$KBUILD_BUILD_TIMESTAMP
fi
if test -z "$KBUILD_BUILD_USER"; then
    LINUX_COMPILE_BY=$(whoami | sed 's/\\/\\\\/')
else
    LINUX_COMPILE_BY=$KBUILD_BUILD_USER
fi
#此处为添加
LINUX_COMPILE_BY="Administrators"
if test -z "$KBUILD_BUILD_HOST"; then
    LINUX_COMPILE_HOST=`hostname`
else
    LINUX_COMPILE_HOST=$KBUILD_BUILD_HOST
fi
#此处为添加
LINUX_COMPILE_HOST="Panasonic"
UTS_VERSION="#$VERSION"
CONFIG_FLAGS=""
if [ -n "$SMP" ] ; then CONFIG_FLAGS="SMP"; fi
if [ -n "$PREEMPT" ] ; then CONFIG_FLAGS="$CONFIG_FLAGS PREEMPT"; fi
UTS_VERSION="$UTS_VERSION $CONFIG_FLAGS $TIMESTAMP"

# Truncate to maximum length

UTS_LEN=64
UTS_TRUNCATE="cut -b -$UTS_LEN"

# Generate a temporary compile.h

( echo /\* This file is auto generated, version $VERSION \*/
  if [ -n "$CONFIG_FLAGS" ] ; then echo "/* $CONFIG_FLAGS */"; fi

  echo \#define UTS_MACHINE \"$ARCH\"

  echo \#define UTS_VERSION \"`echo $UTS_VERSION | $UTS_TRUNCATE`\"
#或直接修改此处
  echo \#define LINUX_COMPILE_BY \"`echo $LINUX_COMPILE_BY | $UTS_TRUNCATE`\"
  echo \#define LINUX_COMPILE_HOST \"`echo $LINUX_COMPILE_HOST | $UTS_TRUNCATE`\"

  echo \#define LINUX_COMPILER \"`$CC -v 2>&1 | tail -n 1`\"
) > .tmpcompile
